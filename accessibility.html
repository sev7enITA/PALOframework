Accessibility Statement

EN 301549 Compliant WCAG 2.1 Level AA Last updated: December 19, 2025

‚úì Compliance Status

PALO Framework is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

Standards Compliance

PALO Framework website and tools conform to:

  • EN 301549 V3.2.1 (2021-03) - European accessibility requirements for ICT products and services
  • Web Content Accessibility Guidelines (WCAG) 2.1 Level AA - W3C Recommendation
  • European Accessibility Act (EAA) - Directive (EU) 2019/882

Accessibility Features

We have implemented the following accessibility features across all pages:

Perceivable

  • Text alternatives for all non-text content (images, icons)
  • Color contrast ratio of at least 4.5:1 for normal text
  • Content can be resized up to 200% without loss of functionality
  • Information is not conveyed by color alone

Operable

  • All functionality available via keyboard navigation
  • Visible focus indicators for keyboard users
  • Skip navigation links on all pages
  • No keyboard traps
  • Sufficient time to complete forms and assessments

Understandable

  • Page language is programmatically determinable (lang="en")
  • Consistent navigation and identification across pages
  • Clear labels and instructions for all form inputs
  • Error messages and suggestions provided

Robust

  • Valid HTML5 markup
  • Proper use of ARIA landmarks and roles
  • Compatible with assistive technologies (screen readers, voice control)

Assistive Technology Compatibility

PALO Framework has been tested with:

  • VoiceOver (macOS/iOS)
  • NVDA (Windows)
  • Keyboard-only navigation
  • High contrast mode
  • Screen magnification (up to 400%)

Known Limitations

We are aware of the following accessibility limitations and are working to address them:

  • Some dynamically generated charts may not be fully accessible to screen readers - we provide alternative text descriptions
  • PDF exports are currently not fully tagged for accessibility - we recommend using the JSON/Markdown export options
